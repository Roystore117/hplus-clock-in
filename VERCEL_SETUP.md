# Vercel デプロイ後のセキュリティ設定手順

このアプリを Vercel に公開した後、**必ず以下の設定をしてください**。
従業員名や店舗名が外部（Google検索など）に漏れるのを防ぐためのものです。

---

## 設定する内容：Password Protection（パスワード保護）

サイト全体に1段パスワードをかけ、知らない人は中身を一切見れないようにします。

### 仕様

- 店舗のiPad/PCで **初回1回だけ** パスワードを入力
- 以降はブラウザに記憶され、自動でログイン状態が維持される（数ヶ月〜1年）
- スタッフは普段通り打刻するだけで、パスワード入力は不要
- Google検索ロボットは完全にブロックされる

---

## 手順

### 1. Vercel にログイン

ブラウザで https://vercel.com を開いてログイン。

### 2. プロジェクトを開く

ダッシュボードから **hplus_clock-in**（または該当するプロジェクト名）をクリック。

### 3. Settings タブを開く

プロジェクトページの上部メニューから **「Settings」** をクリック。

### 4. Deployment Protection を開く

左サイドバーから **「Deployment Protection」** をクリック。

### 5. Password Protection を ON にする

- 「**Password Protection**」セクションを探す
- スイッチを **ON** にする
- 「**Password**」欄に任意のパスワードを入力
  - 例：`tap-in-2026-salon-xxxxx`（推測されにくいものを）
  - **このパスワードは安全な場所にメモしておくこと**
- 「**Save**」をクリック

### 6. 適用範囲を確認

「**Protection Mode**」が以下のいずれかになっていることを確認：

- ✅ `Standard Protection` または `All Deployments` を推奨
  - 本番（Production）も含めて全部に保護がかかる

### 7. 動作確認

1. シークレットウィンドウ（Chromeなら `Ctrl+Shift+N`）でサイトURLを開く
2. パスワード入力画面が表示されたら成功
3. 設定したパスワードを入力 → 打刻画面が表示される

---

## 店舗のiPad/PCでの初期設定

各端末で **1回だけ** 以下を実行：

1. ブラウザでサイトURLを開く
2. Vercel のパスワード入力画面が出る
3. 上で設定したパスワードを入力
4. 「**Remember me**」的なチェックがあれば ON
5. 以降、その端末では自動でログイン状態が維持される

---

## 環境変数の追加設定（**必須**）

`ADMIN_TOKEN_SALT` という環境変数を Vercel と `.env.local` の両方に追加してください。

### なぜ必要か

管理者cookieに保存されるハッシュ値の「ソルト」（混ぜ込む秘密文字列）。これがコードに含まれていると、公開リポジトリ経由で漏洩 → cookie偽造のリスクがあるため、環境変数で管理します。

### 設定方法

#### ローカル開発環境（`.env.local`）

`.env.local` に以下を追記：

```
ADMIN_TOKEN_SALT=<長いランダム文字列>
```

ランダム文字列の生成例（ターミナルで実行）：

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

64文字の16進数文字列が出力されます。それをコピーして使ってください。

#### Vercel 環境変数

1. Vercel管理画面で対象プロジェクトを開く
2. Settings → Environment Variables
3. `ADMIN_TOKEN_SALT` を追加（同じく長いランダム文字列。**ローカルと同じ値である必要はない**）
4. 「Production」「Preview」「Development」全てにチェック → Save
5. Vercel側で再デプロイ（Settingsだけでは反映されないので push か手動 redeploy）

### ⚠️ 既存cookie失効について

`ADMIN_TOKEN_SALT` を変更すると **既存の管理者cookie が全て無効になります**。各端末で再ログインが必要です（オーナー1人なので影響軽微）。

## 既に追加済みの保護（コード側）

以下は既にコードに実装済みで、デプロイすれば自動で有効になります：

| 対策 | 場所 | 効果 |
|------|------|------|
| `X-Robots-Tag: noindex, nofollow` ヘッダー | `next.config.ts` | Google検索結果に載らないよう指示 |
| `robots.txt` でクローラー全拒否 | `public/robots.txt` | クローラーがサイトをクロールしないよう指示 |

これらは「お願いベース」の防御。Vercel Password Protection が「物理的な鍵」。両方あると安心。

---

## 困った時

- パスワードを忘れた → Vercel管理画面の同じ場所で再設定可能
- スタッフから「サイト見れない」と連絡 → パスワードを伝えて1回入力してもらう
- 端末を変えた → 新端末で1回だけパスワード入力が必要
